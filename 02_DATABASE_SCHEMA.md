# 02 — Database Schema

## ไฟล์นี้มีไว้ทำไม

ไฟล์นี้คือ “แบบแปลนฐานข้อมูล” ของ ArtPillPlan v2

Database schema มีไว้บอกว่า:

- ต้องมีตารางอะไรบ้าง
- แต่ละตารางมี field อะไร
- แต่ละ field มีไว้ทำไม
- ตารางไหนเชื่อมกับตารางไหน

ถ้า schema ชัด AI Coding Agent จะสร้าง backend และ UI ได้แม่นขึ้นมาก

# Table: medications

ใช้เก็บรายการยาประจำ

หนึ่ง record = ยา 1 ตัวพร้อมขนาดยา เช่น `Tacrolimus 1 mg`

```sql
create table medications (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  daily_dose_pills numeric(10,2) not null default 1,
  pills_per_box numeric(10,2) not null default 1,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
```

## Fields

| Field | มีไว้ทำไม |
|---|---|
| id | รหัสเฉพาะของยา |
| name | ชื่อยา + ขนาดยา เช่น Tacrolimus 1 mg |
| daily_dose_pills | จำนวนเม็ดที่กินต่อวัน ใช้คำนวณว่ายาเหลือกี่วัน |
| pills_per_box | จำนวนเม็ดต่อกล่องมาตรฐาน ใช้คำนวณราคาต่อกล่อง |
| is_active | ใช้ soft delete ถ้าเลิกใช้ยานี้ |
| created_at | วันเวลาที่สร้าง |
| updated_at | วันเวลาที่แก้ไขล่าสุด |

# Table: stock_lots

ใช้เก็บ Stock แยกตาม Lot

หนึ่ง record = การเพิ่ม Stock 1 ครั้ง

ทุกครั้งที่ Add Stock ต้องสร้าง Lot ใหม่เสมอ

```sql
create table stock_lots (
  id uuid primary key default gen_random_uuid(),
  lot_number bigint generated by default as identity,
  lot_code text unique,
  medication_id uuid not null references medications(id) on delete cascade,
  quantity_pills_original integer not null,
  quantity_pills_remaining integer not null,
  expiry_month integer not null check (expiry_month between 1 and 12),
  expiry_year integer not null check (expiry_year between 2000 and 2100),
  total_price numeric(12,2) not null,
  cost_per_pill numeric(12,4) not null,
  standard_box_price numeric(12,2) not null,
  created_at timestamptz not null default now()
);
```

## Fields

| Field | มีไว้ทำไม |
|---|---|
| id | รหัสเฉพาะของ Lot |
| lot_number | เลข Lot อัตโนมัติ |
| lot_code | แสดงเป็น Lot #001, Lot #002 |
| medication_id | บอกว่า Lot นี้เป็นของยาตัวไหน |
| quantity_pills_original | จำนวนเม็ดตอนเพิ่ม Stock |
| quantity_pills_remaining | จำนวนเม็ดที่เหลือตอนนี้ |
| expiry_month | เดือนหมดอายุ |
| expiry_year | ปีหมดอายุ |
| total_price | ราคาที่จ่ายจริงหรือมูลค่าที่คำนวณได้ |
| cost_per_pill | ราคาต่อเม็ด |
| standard_box_price | ราคาเทียบต่อกล่องมาตรฐาน |
| created_at | วันที่เพิ่ม Stock |

# Table: stock_history

ใช้เก็บประวัติ Stock

เก็บเฉพาะ:

- add_stock
- recount_stock

ไม่ต้องเก็บประวัติแก้ไขข้อมูลยา

```sql
create table stock_history (
  id uuid primary key default gen_random_uuid(),
  medication_id uuid not null references medications(id) on delete cascade,
  stock_lot_id uuid references stock_lots(id) on delete set null,
  type text not null check (type in ('add_stock', 'recount_stock')),
  quantity_pills integer not null,
  price numeric(12,2),
  expiry_month integer,
  expiry_year integer,
  note text,
  created_at timestamptz not null default now()
);
```

## Fields

| Field | มีไว้ทำไม |
|---|---|
| id | รหัสประวัติ |
| medication_id | ประวัตินี้เป็นของยาตัวไหน |
| stock_lot_id | ถ้าเป็น Add Stock ให้เชื่อมกับ Lot |
| type | add_stock หรือ recount_stock |
| quantity_pills | จำนวนเม็ดที่เกี่ยวข้อง |
| price | ราคา ใช้กับ Add Stock |
| expiry_month / expiry_year | วันหมดอายุ ใช้กับ Add Stock |
| note | ข้อความเสริม เช่น Updated to 39 pills |
| created_at | วันเวลาที่เกิดรายการ |

# Table: app_settings

ใช้เก็บค่าตั้งค่าของแอป

เพราะเป็นแอปใช้คนเดียว มี row เดียวก็พอ

```sql
create table app_settings (
  id uuid primary key default gen_random_uuid(),
  low_stock_alert_days integer not null default 14 check (low_stock_alert_days between 1 and 365),
  expiring_lot_alert_days integer not null default 90 check (expiring_lot_alert_days between 1 and 730),
  pin_value text,
  updated_at timestamptz not null default now()
);
```

## Fields

| Field | มีไว้ทำไม |
|---|---|
| low_stock_alert_days | ถ้ายาเหลือน้อยกว่านี้ให้เตือน Low Stock |
| expiring_lot_alert_days | ถ้า Lot จะหมดอายุภายในกี่วันให้เตือน Expiring Lot |
| pin_value | PIN สำหรับ MVP |
| updated_at | วันเวลาที่แก้ไขล่าสุด |

# Recommended Indexes

```sql
create index idx_stock_lots_medication_id on stock_lots(medication_id);
create index idx_stock_lots_expiry on stock_lots(expiry_year, expiry_month);
create index idx_stock_history_medication_id on stock_history(medication_id);
create index idx_stock_history_created_at on stock_history(created_at desc);
```

Indexes มีไว้ให้ค้นหาเร็วขึ้น เช่น โหลด Lot ของยาตัวหนึ่ง หรือโหลด History ล่าสุด
